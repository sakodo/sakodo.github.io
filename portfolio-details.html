Projet Blog

Erika et moi avons choisi d'explorer le thème du cinéma d'horreur pour notre projet de blog. Nous avons décidé d'utiliser un modèle existant et de le personnaliser en utilisant du code HTML, CSS et PHP. Nous sommes impatients de créer un espace en ligne captivant pour partager notre passion pour le cinéma d'horreur avec d'autres passionnés.

Conception d'une Base de Donnée

Mise en place de la Base de donnée

En groupe, en suivant la méthode Merise, nous avons conçu une liste de tables essentielles pour la gestion des données de notre projet. Cela nous a amenés à créer les tables "categories" pour la classification des films, "articles" pour le nom et le contenu des descriptions..., "comments" pour les commentaires, et "users" pour les utilisateurs.

Mise en relation

En établissant un ensemble de valeurs, nous avons pu identifier les contraintes du Modèle Conceptuel de Données (MCD) et élaborer des régles de gestions.

En se fondant sur les cardinalités des entités, nous avons déterminé que
- User (1,n) lire (0,n) articles
- Comments (1,1) lier (0,n) articles
- Article (1,1) avoir (0,n) category

    Mise en place du Code

    J'ai choisi d'adopter l'architecture MVC en PHP afin de garantir une séparation claire des différentes composantes du code. Ainsi, les requêtes sont gérées dans la partie modèle, l'affichage est géré dans les templates de vue, et la logique de contrôle est centralisée dans les contrôleurs situés à la racine du projet.

    Préparation de la requête

    Le fichier articles.model.php du modèle prépare une requête en sélectionnant les propriétés désirées, se connecte à la base de données, puis retourne un tableau associatif dans une variable appelée Result.
